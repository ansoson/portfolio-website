PERSONAS AND STORYBOARDS: (link to deployed page)

The first thing that matters when creating a product is remembering who the product is made for.

In this project, I practiced analysis of user demographics by recording imperative user data through observation, creating visualizations of the average user, and clearly defining the interaction process of those users.

Observation

This project began with observation of a public interface and the interactions that users had with it.

Around 15-20 subjects were observed, and commonalities among them were noted for analysis

Select subjects were interviewed and asked questions about their experience with the interface they used, and thoughts relating to ease of use, efficiency, and emotional state during use.

Personas

Data collected from observation and interview was distilled into hypothesized users, or "personas"

These personas visually and textually depict the thoughts, feelings, words, and actions of one of these users

These personas can then be analyzed for design decisions when updating or creating new products

Storyboards

A persona is depicted going through the actions of using the interface, in order to further visualize and understand the process.

This is done by creating images with text descriptions for each action the user takes in the process

With storyboards created, the entire use process us visualized for easier analysis

RESPONSIVE DESIGN: (link to deployed page)

A design means nothing if you can't also create it.

In this project, I redesigned a website of a local endocrinology firm through two steps of low and high-fidelity prototyping before creating a blueprint for the site in actual html and css

Low fidelity prototyping

Using a low-fidelity sketching program, I created initial rough blueprints for the final design of the site

Designed to take a small amount of time, this prototype would be the building block for the rest of the prototyping stages

High Fidelity Prototyping

Using figma, a finalized prototype was created after revising the initial low fidelity one and using the style guide

This blueprint would then be the basis for creating the final html prototype
